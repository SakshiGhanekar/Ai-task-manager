Project Overview

The NeuraFlow AI Task Management Portal is a comprehensive, production-ready full-stack web application designed to streamline personal and team productivity. It allows users to register securely, manage tasks via a modern Kanban board, and utilize an integrated Google Gemini AI assistant to automatically generate detailed descriptions, assign priorities, and estimate completion times — from just a plain-English task title.

The application features a premium onboarding experience with an animated splash screen, glassmorphism UI design, real-time countdown timers, dark/light mode, and full mobile responsiveness.

  • Register & log in securely with JWT authentication
  • Manage tasks on a Kanban Board (To Do → In Progress → Done)
  • Use Google Gemini AI to auto-generate task metadata
  • Track task progress with real-time color-coded countdowns
  • Visualize analytics on a beautiful Dashboard

AI Workflow
1
User enters task title

User types a task title (e.g. "Fix production crash on file upload") in the task form.

2
Frontend sends POST /api/ai/generate

Axios sends { title: "..." } with the JWT Bearer token in the header.

3
AiService constructs Gemini prompt

The Spring service builds a strict prompt instructing Gemini to return a valid JSON object only.

4
Gemini REST API call via RestTemplate

Spring's RestTemplate dispatches the request to Google's Gemini endpoint with the API key.

5
Response sanitization & parsing

Response is cleaned with regex (strips markdown ```json ... ``` blocks), then parsed by Jackson ObjectMapper into AiGenerateResponse DTO.

6
Form auto-filled ✅

Frontend receives { description, priority, estimatedTime, estimatedHours } and populates the task form instantly.

🔄 Smart Fallback (when API key is missing/invalid)

The backend includes a keyword-detection fallback algorithm that still generates meaningful content:

"bug", "fix", "crash", "error"
→ HIGH priority · Emergency hotfix checklist
"api", "database", "backend"
→ HIGH priority · Backend engineering checklist
"design", "ui", "frontend", "css"
→ MEDIUM priority · UI/UX implementation checklist
"test", "qa", "docs", "update"
→ LOW priority · QA & maintenance checklist

Assignment Document

8.1 Assumptions Made

  • Single-tenant Architecture: The system assumes users manage their own private tasks. Role-based access control is scaffolded but not strictly required for the core task CRUD loop.
  • AI Availability: A valid GEMINI_API_KEY is assumed in the host environment. A smart keyword-based fallback handles API timeouts gracefully.
  • Database Environment: The backend assumes MySQL 8.0+ dialect with DDL auto-create enabled in development.

8.2 Challenges Faced & Solutions

🔴 AI Response Formatting (Markdown Injection)
Language models often inject markdown (e.g., ```json ... ```) into responses. The backend required custom regex sanitization to reliably parse the AI string into a Java DTO before Jackson processing.
🔴 Countdown NaN Bug
Old tasks without a createdAt field caused new Date(undefined) → NaN propagation. Fixed with an isNaN(start) guard — the component now returns null gracefully instead of displaying garbage values.
🔴 UI State Synchronization
Ensuring the UI reacts optimistically to state changes (e.g., instantly updating task status without a full reload) required careful optimistic updates in the Context API with local state mutation before the async API call completes.
🔴 TaskContext createTask Bug
The createTask function referenced undefined local variables (title, description) instead of spreading the taskData parameter. Fixed by replacing with { ...taskData, id: Date.now(), createdAt: new Date().toISOString() }.
